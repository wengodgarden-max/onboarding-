# 学伴（Xueban）· 产品定位与功能设想

> 面向试用期新人的 AI 学习成长伙伴 | V1.0 | 2026-06-02

---

## 一句话定义

**学伴** = 一个比新员工早入职两年、特别懂传音资源的师兄/师姐。

他知道你的情况、他能给建议、他能帮你找资源、他不评判你、他陪你成长。

---

## 解决什么问题？

### 核心矛盾：供给侧和需求侧之间的"转化断层"

```
公司有资料、有导师、有课程 ──❌ 断裂── 新人需要判断支持、需要任务标准、需要求助路径
```

**学伴做的事情：** 把这个断裂接上——把组织已有的资源，转化成新人能直接用、能理解、能消化

### 四个典型卡点（L1-L4 断点）

| 断点 | 新人感受 | 学伴怎么做 |
|------|----------|-----------|
| **L1 学了不会用** | "资料看了，但不知道怎么落地" | 引导描述场景 → 推荐实操课程 → 给行动清单 |
| **L2 目标挂不上** | "老板让我写目标，但我不知道他真正在意什么" | 逐步收集信息 → 生成KPI+重点工作草案 → 导出xHR格式 |
| **L3 任务拆不开** | "拿到一个大任务，不知道从哪开始" | SOP方法论支撑 → 拆成3-5步 → 标注完成标准 |
| **L4 偏了不敢问** | "感觉方向可能不对，但不好意思说" | 校准框架 → 沟通话术 → 自检清单 |

### 调研数据佐证

- **83%** 的新人最迫切需要"关键联系人清单"
- **58%** 认为"信息分散难查找"是最大痛点
- **66%-88%** 基础操作已掌握 → 缺的不是工具培训，**是任务层面的判断支持**

---

## 给谁用？

**V1.0 唯一聚焦：试用期新人（入职 0-90 天）**

典型画像：
- 刚加入传音，对业务流程、团队文化、隐性规则都不熟悉
- 被分配了真实任务，但不知道优先级、完成标准、求助路径
- 有导师，但导师时间不稳定、职责边界不清
- 想表现好但不敢问"蠢问题"

## 能做什么？

### 双线并列：「学」+「创」

```
┌─────────────────────────────────────────────┐
│               学伴（统一入口）               │
├──────────────────┬──────────────────────────┤
│    📖 学（输入）  │     ✨ 创（输出）         │
│                  │                          │
│ ⏰ Timing时机检测 │ 📋 目标制定（含xHR导出） │
│ 📚 知识库推荐    │ 🔨 任务拆解（SOP方法论）  │
│ 🔍 知识发掘      │ 🔄 方向校准（自检框架）  │
│ 🔄 知识沉淀闭环  │ 📊 成果输出（D30/D60/D90）│
└──────────────────┴──────────────────────────┘
```

### 90 天旅程陪伴（6 个关键节点）

| 时间 | 节点 | 做什么 | 产出物 |
|------|------|--------|--------|
| **D1** | 激活 | 欢迎+自我介绍 | 人脉地图 + 2分钟展示稿 |
| **D3** | 目标对齐 | 信息收集 → 个性化目标 | KPI指标 + 重点工作任务 + xHR格式 |
| **D7** | 进展检查 | 主动询问目标推进是否遇到困难 | 卡点识别 + 拆解建议 |
| **D30** | 成果卡 | 月度复盘 | 成果卡 + 进度报告 + 汇报稿 |
| **D60** | 局外人报告 | 新鲜视角观察 | 信息断层报告 + 优化建议 |
| **D90** | 一页纸 | 试用期总结 | 成长一页纸 + 知识库条目 |

### 知识缺口自动管理

新人提问 → AI搜索知识库：
- **命中** → 直接返回答案
- **未命中** → 自动记录为"知识缺口" → 管理员在后台看到 → 补充答案 → 转为知识卡片 → 以后新人再问就能匹配到

## 产品理念

| 维度 | 定位 |
|------|------|
| **产品形态** | 对外是一个 Agent（统一入口），对内是一个**产品系统** |
| **与 T-claw 的区别** | T-claw = 秘书（日常事务）；学伴 = 教练+师兄（学习成长），互不替代 |
| **主动性** | 三档：① 关键节点主动出现；② 被动响应用户提问；③ 日常隐身陪伴 |
| **"自主"** | 员工有选择权 + Agent 基于数据持续进化 |
